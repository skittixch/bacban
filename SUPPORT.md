# Support

Use the repository issue templates for bugs, feature requests, and agent-loop
questions.

Before asking for help:

1. Confirm Docker Desktop is running.
2. Run `docker compose ps`.
3. Check `http://127.0.0.1:3001/health`.
4. Try the demo seed in `docs/DEMO.md`.
5. Remove private board data and credentials from any shared logs.

For security-sensitive problems, use `SECURITY.md` instead of a public issue.
