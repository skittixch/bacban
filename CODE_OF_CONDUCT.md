# Code of Conduct

This project expects practical, respectful collaboration.

## Standards

- Keep issues and pull requests focused on the work.
- Assume good intent, but support claims with reproducible evidence.
- Do not post private board data, credentials, personal contact details, or
  sensitive workflow payloads.
- Respect local-first safety boundaries and explicit operator approval gates.

## Enforcement

Project maintainers may edit, hide, or remove comments and contributions that
violate these standards or expose private data. Security-sensitive reports
should follow `SECURITY.md`.
